Ship software unattended, with evidence.

[ Enterprise Moat ]
Core OSS
Private providers
Managed intelligence

Open the execution contract. Keep the leverage private.

Core OSS Bach

Bach should be credible as a local-first execution engine. The open core owns the Bachfile syntax, plan-first workflows, Factory daemon, SQLite Backend, public trigger protocol, and normalized evidence contracts.

  • Manual trigger and local queue.
  • Agent templates, Policies, Runs, and Targets.
  • JSON-RPC trigger-provider protocol.
  • Evidence/finding schemas and local exports.

Private moat

High-value network integrations stay private: GitHub Issues, Discord, browser or Atelier intake, hosted evidence sync, managed credentials, organization policy, and premium factory workflow packs.

Atelier database advantage

Atelier stores normalized findings, policy verdicts, failure fingerprints, prompt outcomes, and factory timelines for cross-repo dashboards and self-improvement.

Why this works

The protocol is public and minimal, but the best providers, hosted UX, historical data, dashboards, and automated recommendations are enterprise product surface.

[ Target HCL ]
One Project
Split Bachfiles
Backend default

The root Bachfile composes focused files.

Bachfile
import "Bachfile.agents"
import "Bachfile.deploy"
import "Bachfile.factory"

project "my-repo" {
  root = "."

  backend {
    type = "stdio"
    command = ["bach", "backend", "sqlite"]
    config = { path = ".bach/state.db" }
  }
}

Hard break

project.state is removed. No compatibility shim. New docs and examples use backend.

Implementation reality

The existing .bach/state.db stays. SQLite is exposed through the same stdio Backend Provider protocol as future backends.

Omitting backend resolves to the bundled bach backend sqlite provider with path = ".bach/state.db". The provider protocol is the stable boundary.

[ Factory HCL ]
Manual first
Provider triggers
Named workflows

A daemon lane for shipping.

Bachfile.factory
factory "sldc" {
  repo = "."

  triggers {
    manual {}

    provider "github_issues" {
      command = ["atelier-trigger-github-issues"]
      config = { repo = "owner/repo", labels = ["factory:ship"] }
      route { label = "factory:ship", workflow = "ship" }
    }
  }

  workflow "ship" {
    plan { template = agent_template.planner, requires_approval = true }
    implement { template = agent_template.implementer, policy = policy.implementation_ready }
    merge { template = agent_template.merge, policy = policy.merge_ready }
    deploy "staging" { target = pipeline.deploy_staging }
    verify "staging" { target = group.staging_gate }
  }
}
Factory is not a Target.
Completion derives from phase success.
Deploy and verify call normal Bach targets.
Provider implementations can remain private.

[ Agent Templates ]
Reusable machinery
Not runnable
Generated targets

Templates define how agents run, not what work ships.

Bachfile.agents
agent_template "planner" {
  provider = provider.opencode
  role     = "planner"
  prompt   = prompt.factory_planner

  workspace { mode = "readonly" }
}

agent_template "implementer" {
  provider = provider.opencode
  role     = "implementer"
  prompt   = prompt.implementer

  workspace {
    mode = "clone"
    path = ".bach/agents/${factory.workflow}/${work_item.id}/${workstream.id}"
  }

  git {
    branch = "bach/${factory.workflow}/${work_item.id}/${workstream.id}"
    commit = "required"
  }
}

[ Approval And Deploy ]
CLI approval
Durable evidence
Targets only

Production deploy pauses until explicit approval.

approval gate
bach factory approve sldc <work-item-id> --phase deploy.production
Phase behavior
Decision
Waiting approval
A gated phase moves the work item into waiting_approval; the daemon skips or pauses it.
Approval evidence
Backend records factory, work item, phase, approved time, approver identity when available, and reason.
Deploy semantics
Factory deploy and verify phases call normal Bach targets. No new deploy engine in v1.

Later Atelier UI or Discord buttons can call the same approval pathway. The local CLI gate ships first because it is safe, scriptable, and testable.
