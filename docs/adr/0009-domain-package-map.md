# Domain Package Map

Bachkator should refactor away from a large `internal/build` package toward domain subsystem packages: `internal/model` for shared project/target/input/resource types, `internal/config` for Bachfile loading and validation, `internal/runner` for planning/scheduling/execution/cache/log contracts, `internal/state` for persistence, `internal/quality` for report and gate handling, `internal/plugin` or `internal/graph` for graph-load plugins and affected-target evidence, and `internal/cli` for the CLI contract. The goal is to support parallel feature work across target kinds, config types, runner behavior, quality features, and CLI commands without creating import cycles or a new global hotspot. Package splits should preserve the unified **Target** model and use subsystem registries where extensibility is needed.
